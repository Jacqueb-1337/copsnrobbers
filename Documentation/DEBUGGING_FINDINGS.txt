COPS N ROBBERS - STORE BUTTON FREEZE DEBUGGING
===============================================

SESSION START: October 26, 2025

PROBLEM: Game freezes when clicking Store button (also happens with Singleplayer/Multiplayer)

===============================================
FINDINGS - NARROWING DOWN THE FREEZE LOCATION
===============================================

1. INITIAL STATE:
   - Original APK from internet also freezes
   - FMOD audio errors present (7 corrupted files) - user fixed with dummy audio, still freezes
   - Freeze is NOT caused by audio issues

2. BUTTON CLICK SEQUENCE:
   - DefaultPreviewButton.OnClick() → logs added, but never appears in logcat
   - UIMenuDirectorExt.StoreBtnPressed() → logs added, DOES appear
   - UIMenuDirector.ToScene(Store) → logs added, DOES appear
   - Application.LoadLevel("StoreScene") → logs added, DOES appear
   - LoadLevel RETURNS SUCCESSFULLY (confirmed with after-LoadLevel log)
   
3. FREEZE POINT IDENTIFIED:
   ✓ Freeze does NOT happen in button click handler
   ✓ Freeze does NOT happen during scene loading by LoadLevel
   ✗ Freeze DOES happen after LoadLevel returns, during Store scene initialization
   
   Timeline:
   16:32:12.680 - StoreBtnPressed called (log appears)
   16:32:12.680 - About to LoadLevel: StoreScene (log appears)
   16:32:12.683 - LoadLevel returned - scene loaded (log appears)
   [THEN FREEZE - app becomes unresponsive]
   [After ~8 seconds: process killed with signal 9]

4. ROOT CAUSE IDENTIFIED:
   UIStoreDirector.Start() contains 15+ GameObject.Find() calls:
   - cionScrollView = GameObject.Find("View UI(Coins)");
   - cionPanel = GameObject.Find("Panel(Coins)");
   - weaponPanel = GameObject.Find("Panel(Weapon)");
   - scrollView = GameObject.Find("View UI");
   - upgradeTipsPanel = GameObject.Find("Panel(UpgradeTips)");
   - coinPageBtn = GameObject.Find("Image Button(CoinPage)");
   - weaponPageBtn = GameObject.Find("Image Button(WeaponPage)");
   - skinPageBtn = GameObject.Find("Image Button(SkinPage)");
   - tipsLabel = GameObject.Find("Label(Tips)");
   - tipsPriceLabel = GameObject.Find("Label(TipsPrice)");
   - getCoinsBtn = GameObject.Find("Image Button(GetCoins)");
   - cancelBtn = GameObject.Find("Image Button(Cancel)");
   - continueBtn = GameObject.Find("Image Button(Continue)");
   - tipsWindowTop = GameObject.Find("Panel(TipsWindowTop)");
   - tipsWindowBottom = GameObject.Find("Panel(TipsWindowBottom)");
   - toolPanelBg = GameObject.Find("Sprite (ToolBlackStrip)");
   - skinPanel = GameObject.Find("Panel(Skin)");
   
   PROBLEM: GameObject.Find() is O(n) - scans entire scene hierarchy
   ISSUE: Calling it 15+ times in rapid succession causes severe lag/freeze
   
5. CODE PATH:
   UIStoreDirector.Awake() - sets mInstance, returns quickly
   UIStoreDirector.Start() - begins execution, logs don't appear (frozen in the middle)
   → The freeze occurs during those GameObject.Find() calls

===============================================
Logs that appeared:
✓ StoreBtnPressed called
✓ About to LoadLevel: StoreScene  
✓ LoadLevel returned - scene loaded

Logs that DID NOT appear:
✗ UIStoreDirector.Awake() START
✗ UIStoreDirector Awake() - mInstance set
✗ Start() - before GameObject.Find calls
✗ Start() - after GameObject.Find calls

CRITICAL FINDING: UIStoreDirector.Awake() is NEVER CALLED
This means the UIStoreDirector component does NOT EXIST on the StoreScene

ROOT CAUSE ANALYSIS:
The UIStoreDirector MonoBehaviour is not instantiated on the StoreScene GameObject.
Possible reasons:
1. The scene data (*.asset files) in the APK are corrupted or incomplete
2. The GameObject reference in the scene is broken
3. The StoreScene loads but UIStoreDirector component is missing/disabled
4. Some other component's Start() is crashing, preventing the scene from fully initializing

IMPORTANT: The freeze is NOT in UIStoreDirector code - it's during StoreScene initialization itself.
The app freezes AFTER Application.LoadLevel returns but BEFORE any components initialize.
- LoadLevel returns successfully
- Scene loads successfully
- But Start() method never executes or crashes immediately
- Freeze happens between scene load and Start() method execution

NEXT POSSIBILITIES:
1. Another MonoBehaviour on the Store scene is freezing in Awake()
2. Scene initialization itself is hanging
3. UIStoreDirector.Start() crashes on first line before our log
