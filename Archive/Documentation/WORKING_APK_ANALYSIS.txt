=== WORKING APK PACKAGE ID BEHAVIOR ANALYSIS ===

MYSTERY EXPLAINED: Why the working APK installed twice with different package IDs

1. APK MANIFEST PACKAGE ID
   Location: AndroidManifest.xml, line 1
   Value: hd.com.joydo.minestrikenew
   
   This is the OUTER wrapper app - a launcher/installer shell

2. INNER APP PACKAGE ID
   Location: res/values/strings.xml, line 88
   Value: <string name="package_inner_app">com.joydo.minestrikenew</string>
   
   This is the REAL game app that gets installed

=== WHAT HAPPENED WHEN YOU INSTALLED IT ===

Step 1: Initial Installation
   - Android installed the APK with manifest package: "hd.com.joydo.minestrikenew"
   - This appeared as a launcher app in your VPhoneOS
   - This is the outer wrapper/installer application

Step 2: Launch the Wrapper App
   - When you opened the wrapper app (the one with wrong package ID)
   - It read the strings.xml value: package_inner_app = "com.joydo.minestrikenew"
   - The wrapper app then extracted and installed the REAL game with the correct package ID

Step 3: Cascade Installation Triggered
   - The launcher/wrapper detected it wasn't the real game
   - It initiated a download/installation of the proper APK with package: com.joydo.minestrikenew
   - This installed the actual Cops n Robbers game app

Step 4: Wrapper Disappeared but Wasn't Uninstalled
   - The wrapper is still installed (package: hd.com.joydo.minestrikenew)
   - But it hid itself from the launcher after successfully installing the inner app
   - This is common behavior for installer wrappers:
     * They appear in app management settings
     * But hide themselves from the launcher once job is complete
     * Some installer wrappers don't uninstall themselves (they just go dormant)
   - The wrapper might get triggered again if you uninstall the main app

=== TECHNICAL EXPLANATION ===

This is a DUAL-PACKAGE APK architecture, used by some developers to:

1. Distribute a lightweight installer/launcher wrapper
2. Have the wrapper download/install the full game
3. Keep the wrapper installed for future updates/re-installations
4. Avoid re-distributing massive game files each update

The working APK uses this pattern:
   Wrapper (hd.com.joydo.minestrikenew) 
      ↓ launches
      ↓ reads package_inner_app from strings.xml
      ↓ triggers installation of inner app
   Real Game (com.joydo.minestrikenew) ← What you want to run
      ↓ launches successfully
      ↓ NO FREEZES!
