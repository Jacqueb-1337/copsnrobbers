# Cops & Robbers Game - Class Documentation

## Overview
This document provides a structured reference for the major classes in the Cops & Robbers game's decompiled Assembly-CSharp. Classes are organized by functionality.

---

## NETWORKING CLASSES

### PhotonNetwork (Static)
**Type:** Photon Networking Manager (Static Class)  
**Purpose:** Central networking hub for connecting to Photon servers, managing player sessions, and handling multiplayer state.  
**Key Methods:**
- `Connect(serverAddress, port, appId, gameVersion)` - Connects to a custom server
- `Disconnect()` - Closes connection
- `JoinRoom()` / `CreateRoom()` - Room management
- `SendEvent()` - Broadcasts custom events to other players

**Key Properties:**
- `connected` - Whether player is connected to network
- `connectionState` - Current connection status (Connecting, Connected, Disconnecting, etc.)
- `room` - Current Room object (null if not in room)
- `player` - Local player's PhotonPlayer object

**Related Classes:** NetworkingPeer, PhotonPlayer, Room, ServerSettings

---

### NetworkingPeer
**Type:** Core Networking Manager  
**Purpose:** Low-level networking implementation handling all Photon protocol operations and state synchronization.  
**Key Methods:**
- `SendOutgoingStream(stream)` - Serializes and sends game state
- `OnSerializePhotonStream()` - Deserializes incoming state from remote players
- `SendCustomEvent()` - Sends custom RPC calls

**Key Properties:**
- `mCurrentGame` - Reference to current Room
- `mLocalActor` - Local PhotonPlayer
- `State` - PeerState enum (Disconnected, Connecting, Connected, etc.)

**Dependencies:** PhotonNetwork, Room, PhotonPlayer

---

### PhotonView
**Type:** Component (MonoBehaviour)  
**Purpose:** Synchronizes GameObject state across network. Attaches to any object that needs network replication.  
**Key Methods:**
- `OnSerialize(stream, info)` - Called when serializing state
- `OnPhotonSerializeView(stream, info)` - Deserializes incoming state
- `RPC(methodName, targets, params)` - Remote procedure call

**Key Properties:**
- `viewID` - Unique identifier for this view
- `owner` - PhotonPlayer who owns this object

**Related Classes:** PhotonPlayer, PhotonNetwork, Room

---

### PhotonPlayer
**Type:** Data Class  
**Purpose:** Represents a player in the network session.  
**Key Methods:**
- `Equals()` - Compare players
- `ToString()` - Debug representation

**Key Properties:**
- `ID` - Unique player ID
- `Name` - Player's nickname
- `IsLocal` - Whether this is the local player
- `IsInactive` - Whether player has disconnected but object persists

**Related Classes:** PhotonNetwork, Room

---

### Room
**Type:** Data Class  
**Purpose:** Represents a game session/room that players join.  
**Key Properties:**
- `Name` - Room identifier
- `Players` - Array of PhotonPlayers in room
- `MaxPlayers` - Player capacity
- `IsOpen` - Whether room accepts new players
- `CustomProperties` - Room-specific data (map, mode, etc.)

**Related Classes:** PhotonNetwork, PhotonPlayer, RoomInfo

---

### RoomInfo
**Type:** Data Class  
**Purpose:** Lightweight info about a room available in lobby.  
**Key Properties:**
- `Name` - Room name
- `PlayerCount` - Current player count
- `MaxPlayers` - Room capacity
- `RemovedFromList` - If true, room is closing

**Related Classes:** Room, PhotonNetwork

---

### ServerSettings
**Type:** ScriptableObject Configuration  
**Purpose:** Stores Photon server configuration.  
**Key Properties:**
- `ServerAddress` - IP/hostname of Photon server (can be overridden for LAN)
- `Port` - Connection port (default 5055 for UDP)
- `AppID` - Photon application identifier

**Related Classes:** PhotonNetwork

---

### PhotonHandler
**Type:** Component (MonoBehaviour)  
**Purpose:** Provides update loop integration for Photon networking.  
**Key Methods:**
- `OnPhotonSerializeView()` - Core networking update

## PATHFINDING CLASSES (A* Algorithm)

### AstarPath (Static + Component)
**Type:** Pathfinding Manager (MonoBehaviour)  
**Purpose:** Central A* pathfinding system. Manages graph data and path calculation threads.  
**Key Methods:**
- `Scan()` - Build/rebuild navigation graph from level geometry
- `StartPath(path)` - Queue a path for calculation
- `WaitForPath(path)` - Blocking wait for path result
- `GetNearest(position)` - Find closest node to position
- `UpdateGraphs()` - Update graph after level changes

**Key Properties:**
- `graphs` - Array of NavGraphs in scene
- `active` - Static reference to active AstarPath instance
- `heuristic` - Search heuristic (Euclidean, Manhattan, etc.)

**Related Classes:** Seeker, AIPath, NavGraph, Node

---

### AIPath (Component)
**Type:** AI Movement (MonoBehaviour)  
**Purpose:** Generic AI character that follows paths generated by Seeker.  
**Key Methods:**
- `MovementUpdate()` - Calculates desired movement for this frame
- `RotateTowards(dir)` - Rotate character toward direction

**Key Properties:**
- `target` - Target transform to move toward
- `speed` - Movement speed
- `canSearch` - Whether to request new paths
- `canMove` - Whether to execute movement
- `TargetReached` - True when at target position

**Related Classes:** Seeker, RVOController, CharacterController

---

### Seeker (Component)
**Type:** Pathfinding Requester (MonoBehaviour)  
**Purpose:** Interface for requesting and processing paths.  
**Key Methods:**
- `StartPath(start, end, callback)` - Request path from A to B
- `PostProcess(path)` - Run modifiers on completed path
- `RunModifiers()` - Apply path smoothing/modification

**Key Properties:**
- `pathCallback` - Delegate called when path completes
- `lastCompletedVectorPath` - Most recent path waypoints

**Related Classes:** AIPath, AstarPath, Path modifiers

---

### NavmeshController (Component)
**Type:** Navmesh Movement (MonoBehaviour)  
**Purpose:** Constrains movement to valid navmesh surface.  
**Key Methods:**
- `SimpleMove(direction)` - Apply character movement with navmesh clamping
- `ClampToNavmesh(target)` - Ensure position stays on navmesh
- `ClampAlongNavmesh()` - Find valid movement along mesh

---

### Node
**Type:** Pathfinding Node  
**Purpose:** Individual grid/mesh node representing traversable space.  
**Key Methods:**
- `Open()` - Add neighbors to open set during A* search
- `UpdateG()` - Update movement cost from start
- `UpdateH()` - Update heuristic estimate to goal
- `FloodFill()` - Mark connected walkable nodes with same area

**Key Properties:**
- `position` - 3D position (Int3 format)
- `walkable` - Can this node be traversed
- `penalty` - Movement cost multiplier
- `area` - Connected region identifier
- `connections` - Array of neighbor nodes

**Related Classes:** AstarPath, NavGraph

---

## COLLISION AVOIDANCE CLASSES (RVO)

### RVOController (Component)
**Type:** RVO Agent (MonoBehaviour)  
**Purpose:** Implements Reciprocal Velocity Obstacles collision avoidance.  
**Key Methods:**
- `Update()` - Update velocity based on neighbors and obstacles
- `SetTarget()` - Set desired movement direction

**Key Properties:**
- `maxSpeed` - Maximum velocity
- `neighbourDist` - How far to look for collisions
- `Velocity` - Current velocity
- `Position` - Current position

**Related Classes:** RVOSimulator, RVOObstacle, AIPath

---

### RVOSimulator
**Type:** RVO System Manager (MonoBehaviour)  
**Purpose:** Manages all RVO agents and performs collision simulation.  
**Key Methods:**
- `AddAgent(position)` - Register new RVO agent
- `RemoveAgent(agent)` - Unregister agent
- `Update()` - Simulate collision avoidance for all agents

**Related Classes:** RVOController, RVOObstacle, KDTree

---

### RVOObstacle (Abstract Base)
**Type:** Static Obstacle Representation  
**Purpose:** Represents static geometry that agents avoid.  
**Key Methods:**
- Subclasses: RVOSquareObstacle, RVONavmesh

**Related Classes:** RVOSimulator, RVOController

## SUMMARY

The Cops & Robbers game is built on:
1. **Photon Networking** - Multiplayer synchronization
2. **A* Pathfinding** - NPC movement planning
3. **RVO Collision Avoidance** - Smooth multi-agent interaction
4. **Custom Gameplay Logic** - Cops vs Robbers mechanics
5. **NGUI** - User interface management
6. **Visual Effects System** - Explosions and impacts

Key integration points for modifications:
- Redirect Photon server: Modify `ServerSettings.ServerAddress` or use `PhotonNetwork.Connect()`
- Add new weapons: Extend `Gun` class and register in `WeaponController`
- Create new maps: Add to `AstarPath` graphs and define play areas
- Implement new game modes: Create variant of `Game` manager
